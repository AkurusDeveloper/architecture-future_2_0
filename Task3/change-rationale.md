# Обоснование изменений

## Этап 1: Портал самообслуживания и IdP

**Технологии:** Портал витрины данных, IdP, каталог данных, облачные витрины

**Бизнес-эффекты:**
- Скорость принятия решений: отчеты готовятся за минуты вместо часов
- Качество сервиса: self-service снижает нагрузку на IT
- Снижение затрат: меньше запросов на ручную подготовку отчетов
- Регуляторика: единый вход упрощает аудит доступа

**Связка:** Облачные витрины → быстрый доступ к данным → сокращение времени отчетности с часов до минут

## Этап 2: Облачный DWH и оркестрация ETL

**Технологии:** Облачный DWH, Airflow, Data Lake, перенос логики в домены

**Бизнес-эффекты:**
- Независимость направлений: домены развиваются автономно
- Масштабирование: облачный DWH масштабируется по требованию
- Снижение затрат: отказ от поддержки железа, оптимизация ресурсов
- Качество сервиса: управляемые пайплайны повышают надежность

**Связка:** Облачный DWH + перенос логики → независимость доменов → ускорение time-to-market новых продуктов

## Этап 3: Платформа потоков и интеграции

**Технологии:** Kafka, Schema Registry, Flink, событийная архитектура

**Бизнес-эффекты:**
- Масштабирование экосистемы: новые партнеры подключаются без изменения ядра
- Независимость направлений: слабая связность через события
- Скорость принятия решений: потоковая обработка ускоряет аналитику
- Качество сервиса: изоляция доменов повышает отказоустойчивость

**Связка:** Kafka + события → decoupling доменов → подключение партнера за дни вместо месяцев

## Этап 4: Каталог данных и витрины BI

**Технологии:** Каталог данных, расширенные витрины, Data Quality тесты, Observability

**Бизнес-эффекты:**
- Регуляторика: прозрачность владения данными, соответствие требованиям
- Качество сервиса: раннее обнаружение проблем, снижение MTTR
- Скорость принятия решений: качественные данные ускоряют аналитику
- Снижение затрат: автоматизация тестирования данных

**Связка:** Каталог данных + ролевой доступ → прозрачность → соответствие регуляторам и снижение рисков

## Ключевые связки технологий и бизнес-сценариев

1. **Облачный DWH → быстрая отчетность:** Масштабирование по требованию сокращает время расчетов с часов до минут

2. **Событийная архитектура → независимость доменов:** Слабая связность позволяет развивать финтех и ИИ автономно

3. **Платформа потоков → подключение партнеров:** Новые партнеры подключаются через интеграционный слой без изменения ядра

4. **Observability → надежность:** Централизованный мониторинг снижает MTTR и повышает доступность критичных систем

5. **Каталог данных → регуляторика:** Прозрачность владения данными и ролевой доступ обеспечивают соответствие требованиям

6. **Data Quality тесты → качество данных:** Раннее обнаружение проблем в пайплайнах повышает доверие к данным
