# Домены и границы

## 1. Домен Клиник

**Ответственность:** Управление медицинскими процессами, пациентами и клиническими данными.

**Данные/агрегаты:** Медицинские карты, истории болезней, результаты исследований, расписания, персонал клиник, инвентарь.

**Бизнес-сценарии:** Прием пациентов, ведение медкарт, назначение лечения, учет персонала и оборудования.

**Связи:** Публикует события операционных данных (обезличенные для аналитики) в интеграционный слой. Не передает содержимое медкарт в витрину данных.

## 2. Домен Финтех

**Ответственность:** Банковские продукты, платежи, финансовые транзакции.

**Данные/агрегаты:** Счета, транзакции, кредитные продукты, платежи за медуслуги, финансовые отчеты.

**Бизнес-сценарии:** Обработка платежей, выдача кредитов, управление банковскими продуктами, финансовая отчетность.

**Связи:** Публикует события финансовых транзакций и продуктов. Изолирован от клинического контура. Имеет выделенные витрины для отчетности.

## 3. Домен ИИ-медицины

**Ответственность:** Обработка медицинских данных с помощью ИИ, исследовательские задачи.

**Данные/агрегаты:** Обезличенные медицинские наборы, модели ИИ, результаты анализа, исследовательские данные.

**Бизнес-сценарии:** Анализ медданных, разработка моделей, исследовательские проекты, сервисы на основе ИИ.

**Связи:** Получает согласованные наборы данных через платформу данных. Публикует результаты анализа. Использует отдельные пайплайны и песочницы.

## 4. Домен Партнеров

**Ответственность:** Интеграция с внешними партнерами (фармацевтика, производители медоборудования).

**Данные/агрегаты:** Данные о препаратах, оборудовании, поставках, контракты.

**Бизнес-сценарии:** Управление поставками, интеграция каталогов, обмен данными с партнерами.

**Связи:** Подключается через интеграционный слой без изменения ядра платформы. Публикует события о поставках и контрактах.

## 5. Домен Платформы данных

**Ответственность:** Оркестрация ETL, хранение сырых данных и витрин, управление пайплайнами.

**Данные/агрегаты:** Сырые наборы в объектном хранилище, витрины и агрегаты в облачном DWH, метаданные.

**Бизнес-сценарии:** Загрузка данных, трансформации, формирование витрин, снижение времени подготовки отчетов.

**Связи:** Получает события из интеграционного слоя. Формирует витрины для аналитики. Предоставляет метаданные каталогу.

## 6. Домен Интеграции

**Ответственность:** Коннекторы, адаптеры, платформа потоков, валидация контрактов.

**Данные/агрегаты:** События доменов, схемы сообщений, контракты интеграций.

**Бизнес-сценарии:** Подключение источников, публикация событий, валидация данных, изоляция доменов.

**Связи:** Принимает данные от доменов. Публикует события в платформу потоков. Обеспечивает слабую связность между доменами.

## 7. Домен Витрины данных

**Ответственность:** Портал самообслуживания, сервис отчетности, BI-визуализация.

**Данные/агрегаты:** Готовые отчеты, дашборды, витрины для аналитики (без медкарт).

**Бизнес-сценарии:** Самообслуживание аналитикой, генерация отчетов, управленческая аналитика.

**Связи:** Читает из DWH витрины и агрегаты. Использует каталог для поиска данных. Проверяет доступ через управление доступом.

## 8. Домен Доступа и каталога

**Ответственность:** Единая идентификация, роли, каталог данных, управление доступом, аудит.

**Данные/агрегаты:** Пользователи, роли, права доступа, метаданные наборов, журналы аудита.

**Бизнес-сценарии:** Аутентификация, авторизация, поиск данных, контроль доступа, соответствие регуляторам.

**Связи:** Синхронизируется с IdP. Предоставляет метаданные витрине. Управляет доступом к данным.
