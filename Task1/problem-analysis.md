# Анализ проблемных мест

| ID | Проблема | Где проявляется | Затронутые бизнес-сценарии | Комментарии/риски |
|----|----------|-----------------|---------------------------|-------------------|
| P1 | Медленная подготовка сложных отчетов | DWH MS SQL 2008, Power BI | Быстрая подготовка отчетности и управленческая аналитика | Построение отчетов занимает часы, блокирует принятие решений |
| P2 | Зависимость от централизованного DWH | Монолитный DWH, бизнес-логика в хранилище | Независимое развитие финтех и ИИ направлений | Изменения в одном направлении затрагивают другие домены |
| P3 | Легаси технологии затрудняют развитие | MS SQL 2008, PowerBuilder, Apache Camel | Скорость вывода новых продуктов (time-to-market) | Низкая производительность, высокие затраты на поддержку |
| P4 | Перегрузка трансформаций | DWH, ETL процессы | Масштабирование под новые бизнесы/партнеров | Рост объемов данных (сотни ТБ) приводит к деградации |
| P5 | Сложности интеграции новых направлений | Монолитный DWH, интеграционная шина | Масштабирование под новые бизнесы (фарма, медоборудование) | Невозможность гибко подключать партнеров без изменения ядра |
| P6 | Узкое место интеграционная шина | Apache Camel, отсутствие изоляции | Независимое развитие доменов | Отсутствие изоляции по доменам, единая точка отказа |
| P7 | Ограниченная отказоустойчивость | Монолитная архитектура | Надежность и доступность (SLA) | Нет резервирования, высокие операционные риски |
| P8 | Требования регуляторов к защите данных | Отсутствие разделения медицинских и финансовых данных | Соответствие регуляторам (медицинским/финансовым) | Риски несоответствия требованиям защиты данных |
| P9 | Разнородные стеки усложняют доставку | Финтех и ИИ используют разные технологии | Качество медуслуг/финуслуг | Сложность единого процесса доставки данных |
| P10 | Отсутствие каталога данных и управления доступом | Нет централизованного каталога | Соответствие регуляторам, качество сервисов | Нет прозрачности владения данными, сложный контроль доступа |
